监听

server.listen实际上有三个参数

  1. 端口port
  2. 主机host ，通常可以传入localhost，ip地址127.0.0.1，ip地址0.0.0.0（默认值）
    • localhost：本质上是一个域名，通常会被解析为127.0.0.1
    • 127.0.0.1：回环地址，意思是逐级自己发出去的包直接被自己接收
      • 正常的数据包经过 应用层-传输层-网络层-数据链路层-物理层
      • 但回环地址在网络层就直接被获取到了，不经过数据链路层和物理层
    • 0.0.0.0：监听IPV4上所有的地址，再根据端口找到不同的应用程序
  3. 回调函数，服务器成功启动时的回调函数

在浏览器中访问时浏览器本身除了直接访问一次以外，还会访问一次当前端口下/favicon.ico地址，所以会看到两次请求。

request对象

本质上是一个可读流

在向服务器请求时会携带很多信息，比如URL、请求方式、headers等等都会封装在request对象中。

query参数

通过request.url可以获取URL，通过内置模块 url 的方法 parse 可以获取 query 字符串，内置的queryString模块可以将query字符串转换为对象

body参数

因为request参数是一个可读流，所以想要获取请求中的body参数，想要监听data事件，注意，response.end只能在request流end事件触发后进行

headers

content-type

包含了这次请求携带的数据类型

  • application/x-www-from-urlencoded:表示数据被编码为以’&’分割的键值对，同时以’=’分割键与值
  • application/json:表示是一个json类型
  • text/plain:表示是文本类型
  • application/xml:表示是xml类型
  • mutipart/form-data:表示上传文件

content-length

表示数据的总长度

keep-alive

http是基于TCP协议的，但通常在一次请求和响应结束后就会立即中断连接，

http1.0中，如果想要继续保持连接，需要

  • 浏览器在请求头中添加 connection：keep-alive；
  • 服务器需要在响应头中添加 connection：keep-alive；
  • 客户端再次请求时就可以使用同一个连接，直到一方中断连接；

http1.1中，所有的连接默认是 connectiion：keep-alive的；

  • 但不同web服务器会有不同的保持 keep-alive 的时间；
  • Node中默认是5s

accept-encoding

告诉服务器，客户端支持的文件压缩格式，比如js文件可以使用gzip编码，对应.gz文件

accept

告诉服务器，客户端可以接受的文件格式类型

user-agent

客户端相关的信息

authorization

客户端携带的token

response

本质上是一个可写流

如果希望给客户端响应结果数据，可以通过两种方式：

  • write方法：直接写入数据，可以多次写入，不关闭流
  • end方法：写入最后的数据，最后关闭流

如果始终没有调用这两者，客户端会一直等待

  • 所以客户单一般会设置超时时间

状态码

状态码是用来表示http响应状态的数字代码

在response.stateCode中设置

或者通过response.writeHead()方法的第一个参数设置

header

通过设置响应头告知客户端获取的信息编码格式等等，

或者与状态码一起设置
