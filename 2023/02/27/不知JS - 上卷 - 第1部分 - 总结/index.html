变量查询

当JS引擎执行变量赋值操作时，会进行两种变量查询，LHS（左值查询）与 RHS（右值查询），两种查询都会先在当前定义域内查询这个变量是否存在，找到的话就返回这个变量，找不到就去上一层继续查，直到来到最顶层（全局作用域，在浏览器中可以通过 window 对象直接访问，Nodej.s中则是 global 对象）查询，然后结束查询。

LHS（左值查询）

之所以叫这个名字是因为它的作用对象通常出现在 = 的左侧。JS引擎会逐级查询，如果在全局作用域仍然找不到，一般情况下会在全局作用域中创建这个变量并返回，严格模式下，则会抛出 ReferenceError 异常。

值得注意的是，此处查询与返回的目标仅仅是变量本身，而非这个值，可以认为是在寻找存放值的容器。

RHS（右值查询）

同样的RHS（右值查询），一般发生在 = 的右侧或者 = 不存在时（比如调用函数时传入的参数，命令行式编程时直接输入变量名查询值等等），逐级查询到全局作用域仍找不到改变量，则直接抛出 ReferenceError 异常。

这种查询与返回的目标为目标变量的值，也就是变量的内容。

对象属性的查询不在此类讨论中。

函数作用域

当你声明一个函数时，会建立一个包含内部所有声明的标识符以及参数名的定义域，外部不能访问这个函数内部声明的标识符和参数。但作用域内仍然可以访问外部作用域，但如果内部作用域中定义了某个变量与外部变量重名，则无法访问外部变量，这是由于RHS查询与LHS查询的方式导致的。

块作用域

ES6性添加的变量定义关键字 let 可以在任意作用域内约束变量（比如单纯的 {} 内）。而传统的var只会被约束在函数作用域内，甚至不能被 if 的作用域约束！
